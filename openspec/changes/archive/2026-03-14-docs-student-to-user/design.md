## Context

The documentation and source comments in two files use the word "student(s)" to describe the target audience. The project serves a broader audience, so this language should be neutralized to "user(s)".

## Goals / Non-Goals

**Goals:**
- Replace "student(s)" with "user(s)" in all documentation and source comments

**Non-Goals:**
- Rewriting surrounding prose or restructuring documentation
- Changing any code behavior

## Decisions

- **Direct text substitution** — Each occurrence is a simple word swap with no surrounding sentence restructuring needed. "so students can start with…" → "so users can start with…" and "for students" → "for users".

## Risks / Trade-offs

No risks. Two isolated word changes with zero functional impact.
