## Why

The spectrogram text art in the Audio Steganography tutorial (Method 3: Spectrogram Text) uses the Viridis colormap with a linear mapping across the full -80 to 0 dB range. This makes the embedded text hard to read at higher frequencies because the magnitude of the rendered tones doesn't fully saturate the bright end of the colorscale — the text blends into the mid-range colors rather than popping against the dark background.

## What Changes

- **Replace Viridis with a higher-contrast colormap** (e.g., `Hot`, `Inferno`, or a custom scale) in the spectext-related spectrogram visualizations, OR apply a non-linear mapping (gamma/power curve) so more of the color range maps to the bright end.
- Changes apply to **docs-only spectrogram HTML** files generated by `gen_signal_plots.c` and the `spectrogram_text.c` example program. Both use Plotly's `colorscale` property and the C-side Viridis LUT for PNG output.
- No changes to the underlying spectrogram generation, STFT parameters, audio synthesis, or library code (`MD_spectrogram_text`, `MD_stft`, etc.).

## Capabilities

### New Capabilities
- `spectext-colormap`: Colormap selection and mapping for spectext spectrogram visualizations (HTML/Plotly and PNG output)

### Modified Capabilities

(none — this is a visualization-only change; no spec-level behavioral requirements change)

## Impact

- **`examples/gen_signal_plots.c`** — Update `colorscale:` in the Plotly config for spectext-related HTML files (`spectext_hello_spectrogram.html`, `steg_spectext_spectrogram.html`)
- **`examples/spectrogram_text.c`** — Update `colorscale:` in `write_html()` and the Viridis LUT / mapping logic in `write_png()`
- **`guides/plots/spectext_hello_spectrogram.html`** and **`guides/plots/steg_spectext_spectrogram.html`** — Regenerated with new colormap
- **Other spectrograms** (DTMF, chirp, etc.) are NOT affected — only spectext spectrograms change
