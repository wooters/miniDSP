# VAD Optimization: uv Project Setup

## Motivation

The `optimize/VAD/` directory contains a Python script (`optimize_vad.py`)
that tunes VAD hyperparameters using Optuna. It depends on several
third-party packages (`pyminidsp`, `optuna`, `numpy`, `scipy`,
`soundfile`). The miniDSP repo is a C library with no existing Python
dependency management. We need a way to declare and install these
dependencies reproducibly without polluting the rest of the repo.

## Approach

Place a self-contained uv project in `optimize/VAD/`. This directory gets
its own `pyproject.toml` and `uv.lock`. No Python configuration exists at
the repo root or in parent directories.

### Why this approach

- **Isolation** -- The optimization script's dependencies don't leak into
  the rest of the C project.
- **Reproducibility** -- `uv.lock` pins exact versions; committed to the
  repo.
- **Low ceremony** -- `uv run python optimize_vad.py ...` handles venv
  creation and dependency installation automatically.
- **Future-proof** -- If other optimization tools appear under `optimize/`,
  each gets its own `pyproject.toml`. No shared dependency conflicts.

### What changes

1. New `optimize/VAD/pyproject.toml` declaring dependencies and
   `requires-python >= 3.10`.
2. New `optimize/VAD/uv.lock` generated by uv (committed).
3. Append `.venv/` to the repo-root `.gitignore`.
4. Update `optimize/VAD/README.md` to show `uv run` commands.
